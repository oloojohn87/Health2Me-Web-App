Golden Rules of Report Access

Important Database tables related to Report viewing


-> lifepin table - This single table contains the entries for all files being entered into Health2Me system. Important fields are Rawimage(which contains the unique identifier for identifying the encrypted file), IdMed and IdCreator.

->doctorslinkusers - This table contain entries which are inserted when a doctor creates a member, a member initiates a telemedicine phone/video consultation and when a member sends a mail to doctors outside of Health2Me network higlighting certain reports.

->doctorslinkdoctors - This table comes into play mostly when referrals are being made to doctors by doctors. Entries are made into the table about the files which are being attached during referral process.

->doctorsgroup - This table contains the group and doctor ids which fall in a group.

Rule 1 (IdCreator Rule)


-> We have a column in lifepin table called Idcreator which implies who created/uploaded the file. All such files which have been created/uploaded by the doctor(Idcreator) such reports should be viewable to the doctor unless it has been blinded by the member.

->

->

Rule 2 (Group Membership Rule)


->Under this rule if a doctor is member of a group then all members which are associated with all the doctors of that group, all such members and their reports are viewable and accesible to a doctor.

->

Rule 3 (Patient Permission Rule)




->Under this rule, a doctor can seek permission from a member to view a report which is locked.

->This can be achieved by clicking on the lock icon on a report thumbnail. On clicking, it prompts the doctor with an alert to seek permission for access from the user/member. If clicked OK, it sends an email to the member stating the request from the doctor.

Rule 4 (Doctor Sharing rule)


->This rule comes into play when a Health2Me doctor referes a member to another Health2Me or non-Health2Me doctor. Under such circumstances during referral, reports of members are highlighted to the doctor and entries made at the backend tables doctorslinkdoctors to these higlighted reports
